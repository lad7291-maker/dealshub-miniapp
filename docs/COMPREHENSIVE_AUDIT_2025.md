# Комплексный аудит SmartSkidka.ru

**Дата аудита:** 14.05.2026

---

## Executive Summary

SmartSkidka.ru — статический affiliate-агрегатор скидок AliExpress на чистом HTML/CSS/JS (Vanilla), ~1000 товаров, монетизация через Admitad. Сайт имеет солидную базу: HTTPS, Schema.org, Open Graph, sitemap.xml, PWA-манифест, двойную аналитику (GA4 + Я.Метрика). Однако есть критические узкие места: отсутствие кэширования данных (870KB грузятся каждый раз), отключённый Service Worker, слабая внутренняя перелинковка, отсутствие категориальных страниц и недостаточная аналитика событий. Главная возможность — внедрение "быстрых побед" даст заметный рост Core Web Vitals, SEO-трафика и конверсии без серьёзных затрат.

## 1. Технический и PWA-аудит

### Диагноз

Сайт работает на статике с минимальным бэкендом (Node.js API для AI-поиска), но имеет критический bottleneck — отсутствие кэширования для JSON-данных и отключённый Service Worker, который ломает PWA-функциональность.

### Проблемы

1. **products/all.json (868KB)** отдаётся с Cache-Control: no-cache, no-store, must-revalidate — каждый раз грузится заново. **High** — лишний трафик, медленная загрузка на мобильных.
2. **products-loader.js** добавляет ?v=Date.now() и cache: 'no-store' — отключает любое кэширование. **High** — даже при повторном визите данные грузятся заново.
3. **Service Worker отключён**: код очищает caches и делает unregister() при каждой загрузке. **High** — PWA не работает, нет офлайна, install prompt бесполезен.
4. **Нет gzip/brotli для HTML** (статика CSS/JS сжата, HTML — нет). **Medium** — HTML 17.8KB можно сжать до ~5KB.
5. **products.js (870KB)** загружается в head синхронно, блокируя рендер. **Medium** — хотя defer есть, inline-скрипт PRODUCTS_DB может тормозить.
6. **Нет loading="lazy"** на изображениях в карточках товаров (только в item/). **Medium** — все картинки грузятся сразу.
7. **Нет fetchpriority, preload** для критических ресурсов. **Low** — можно улучшить LCP.
8. **PWA install prompt** есть в DOM, но без работающего SW — не функционирует. **Medium** — пользователь видит UI, но установка невозможна.

### Рекомендации

- **High / Low**: Убрать cache: 'no-store' и ?v=Date.now() из products-loader.js, заменить на разумное кэширование (1 час для данных)
- **High / Low**: Настроить nginx: Cache-Control: public, max-age=3600 для products/\*.json
- **High / Low**: Включить gzip/brotli для HTML в nginx
- **High / Medium**: Вернуть Service Worker с кэшированием статики (после тестирования) или удалить dead code очистки кэша
- **Medium / Low**: Добавить loading="lazy" на все img в карточках товаров
- **Medium / Low**: Добавить fetchpriority="high" для первого изображения hero
- **Low / Medium**: Рассмотреть разбиение products.js на чанки по категориям

## 2. SEO-аудит

### Диагноз

Базовая SEO-структура на месте, но отсутствует внутренняя перелинковка, нет категориальных страниц и breadcrumb-навигации. Карточки товаров хорошо оптимизированы, но главная теряет трафик по категориям.

### Проблемы

1. **Нет страниц категорий** (/electronics.html, /clothing.html и т.д.). **High** — теряется трафик по "скидки электроника aliexpress".
2. **Нет breadcrumb-навигации** на item/ страницах. **Medium** — хуже понимание структуры поисковиками.
3. **Нет BreadcrumbList** в Schema.org. **Medium** — недостаёт rich snippet.
4. **На item/ страницах нет h2/h3**, только h1. **Medium** — плохая структура контента.
5. **SEO-блок на главной находится после футера** (ниже видимой области). **Medium** — поисковики могут считать его менее важным.
6. **Нет внутренних ссылок** между товарами ("похожие", "с этим покупают"). **Medium** — плохой link juice distribution.
7. **Нет og:locale, og:site_name**. **Low** — неполнота Open Graph.
8. **Нет страниц /blog/ или /guides/** для информационного трафика. **Medium** — упущенный long-tail.
9. **robots.txt запрещает /_?_** — корректно, но проверить, не режет ли полезные параметры. Low.
10. **sitemap.xml содержит 1005 URL**, но нет категориальных страниц. Low.

### Рекомендации

- **High / Medium**: Создать страницы категорий: /electronics.html, /clothing.html, /home.html, /auto.html и т.д.
- **High / Medium**: Добавить breadcrumb на item/ страницы + BreadcrumbList в Schema.org
- **Medium / Low**: Добавить h2/h3 на item/ (например, "Описание", "Характеристики", "Похожие товары")
- **Medium / Low**: Переместить SEO-блок выше (перед футером, но после товаров)
- **Medium / Medium**: Добавить блок "Похожие товары" на item/ с 4-8 ссылками
- **Medium / Medium**: Создать 3-5 информационных статей (/blog/ или /guides/) для long-tail SEO
- **Low / Low**: Добавить og:locale: ru_RU и og:site_name: SmartSkidka.ru

## 3. UX/UI и конверсия (CRO)

### Диагноз

Дизайн современный и чистый, но есть критические UX-проблемы, которые снижают доверие и конверсию: вводящая в заблуждение кнопка "В корзину", отсутствие срочности, неработающие категории в футере и скрытый Telegram-канал.

### Проблемы

1. **Кнопка "В корзину"** на карточке товара — вводит в заблуждение (нет корзины, это affiliate). **High** — пользователь ждёт корзину, а попадает на AliExpress.
2. **Нет явного объяснения** модели "партнёрская ссылка — цена не меняется". **High** — недостаток доверия.
3. **Категории в футере** (data-cat) — клики не работают (нет обработчика). **Medium** — сломанная навигация.
4. **Нет таймера скидки** или срочности. **Medium** — нет FOMO, ниже конверсия.
5. **Нет social proof** ("X человек купили сегодня", "популярное"). **Medium** — меньше доверия.
6. **Telegram-канал @SmartRuMarket** скрыт только в contact.html. **Medium** — упущенный канал удержания.
7. **Модальное окно** на главной — дополнительный клик до покупки. **Medium** — friction в воронке.
8. **Нет sticky CTA** на мобильном. **Medium** — на длинных страницах кнопка уходит.
9. **Нет отзывов/фото** от покупателей (только рейтинг с AliExpress). **Low** — но рейтинг уже есть, это хорошо.
10. **Фильтры только по скидке**, нет по категории на главной. **Low** — category tabs отключены в коде.

### Рекомендации

- **High / Low**: Переименовать "В корзину" → "На AliExpress" или "Купить со скидкой"
- **High / Low**: Добавить микротекст под кнопкой: "Партнёрская ссылка. Цена для вас не меняется."
- **High / Low**: Исправить/удалить категории в футере (или добавить обработчик)
- **Medium / Low**: Добавить таймер "Скидка действует ещё X часов" (рандомизированный)
- **Medium / Low**: Добавить social proof: "X человек смотрят этот товар" (рандомизированный)
- **Medium / Low**: Добавить видимую ссылку на Telegram-канал в шапку или футер
- **Medium / Low**: Включить category tabs на главной (код есть, но return; в initCategories)
- **Medium / Medium**: Добавить sticky bottom bar на мобильном с CTA "Купить на AliExpress"
- **Low / Low**: Добавить кнопку "Поделиться" для товаров

## 4. Аналитика и метрики

### Диагноз

Базовая аналитика настроена (GA4 + Я.Метрика), но событийная воронка слабая: нет отслеживания поиска, фильтров, PWA-установок, скролла. Это не позволяет понять, где пользователи отваливаются.

### Текущие события

- **purchase**: GA4 + Я.Метрика reachGoal('purchase') — Кнопка "Купить" на item/ и в модалке
- **view_item**: GA4 + Я.Метрика reachGoal('view_item') — Открытие модального окна
- **add_to_favorites**: GA4 + Я.Метрика reachGoal('add_to_favorites') — Клик по сердечку

### Проблемы / недостающие события

1. **Событие search** (поисковые запросы). **High** — непонятно, что ищут пользователи.
2. **События select_filter, select_sort**. **Medium** — непонятно, какие фильтры популярны.
3. **Событие install_pwa**. **Medium** — не отслеживается установка.
4. **Событие share**. **Low** — если добавить кнопку "Поделиться".
5. **События scroll_25, scroll_50, scroll_75, scroll_90**. **Medium** — нет понимания глубины просмотра.
6. **Событие click_outbound** (клик по партнёрской ссылке из модалки). **Medium** — в модалке нет gtag на кнопке.
7. **Нет User-ID** для связи сессий. **Low** — нет кросс-девайс аналитики.
8. **Нет отслеживания ai_search** (запросы AI-поиска). **Medium** — непонятно, используют ли функцию.

### Рекомендации

- **High / Low**: Добавить gtag('event', 'search', {search_term: query}) при поиске
- **High / Low**: Добавить gtag('event', 'select_content', {content_type: 'filter'/'sort', item_id: ...})
- **Medium / Low**: Добавить события скролла (25/50/75/90%)
- **Medium / Low**: Добавить gtag('event', 'install_pwa') при клике на install-btn
- **Medium / Low**: Добавить gtag на кнопку "Купить" в модальном окне (сейчас только на item/)
- **Medium / Low**: Добавить gtag('event', 'ai_search', {search_term: query})

### Гипотезы A/B-тестов

1. **"В корзину" → "Купить на AliExpress"** увеличит CTR на 15-25%. Мерить: CTR кнопки, достижение purchase.
2. **Добавление таймера скидки** увеличит конверсию на 10-20%. Мерить: purchase на item/ с таймером vs без.
3. **Sticky CTA на мобильном** увеличит конверсию на 20-30%. Мерить: purchase на мобильных.
4. **Social proof** ("X человек смотрят") увеличит доверие и CTR. Мерить: время на странице, purchase.
5. **Прямая ссылка на item/** вместо модального окна ускорит путь до покупки. Мерить: время до purchase, bounce rate.

## 5. Security и базовый risk-аудит

### Диагноз

Базовая защита на уровне: HTTPS, HSTS, CSP, X-Frame-Options, X-Content-Type-Options. Но есть потенциальные риски с XSS через поиск и отсутствие ограничения на размер localStorage.

### Проблемы

1. **CSP есть**, но script-src разрешает 'unsafe-inline'. **Medium** — снижает защиту от XSS.
2. **Поисковый запрос** вставляется в innerHTML через шаблонные строки в createProductCard. **Medium** — потенциальный XSS (хотя данные из контролируемого JSON).
3. **localStorage** для избранного без проверки размера/лимита. **Low** — теоретически может переполниться.
4. **API /api/ai-search** открыт без авторизации (только rate limit по IP). **Low** — текущий rate limit 3/24ч достаточен.
5. **Нет Referrer-Policy** в заголовках? **Low** — проверить, в HTTP-заголовках есть.

### Рекомендации

- **Medium / Low**: Экранировать поисковый запрос перед вставкой в HTML (или использовать textContent)
- **Medium / Low**: Добавить лимит на размер избранного (например, 100 товаров)
- **Low / High**: Убрать 'unsafe-inline' из CSP (сложно, т.к. много inline-скриптов)
- **Low / Medium**: Добавить CAPTCHA или усилить rate limit, если снимать ограничение AI-поиска
