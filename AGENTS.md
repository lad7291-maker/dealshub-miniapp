# SmartSkidka.ru — Agent Guide

## Архитектура системы

SmartSkidka.ru состоит из двух проектов:

- **Website**: `/var/www/dealshub-miniapp` — React 19 + Vite + Tailwind CSS, статическая генерация
- **Agents**: `/opt/smart-skidka-agents` — Python оркестратор с 7 типами агентов

## Фиды (3 штуки)

Все фиды генерируются при сборке v2:

1. **sitemap.xml** — карта сайта для поисковиков (1023 URL)
2. **turbo.xml** — Yandex Turbo RSS (топ 100 товаров, рейтинг ≥4.0, скидка ≥50%)
3. **yandex-market.yml** — Yandex Market YML (1000 офферов)

## Источник данных

- **Admitad XML feed** — URL в переменной окружения `ADMITAD_XML_URL`
- **Обновление**: запускается **feed-agent** (LLM-агент с интеллектуальным анализом)
- **Расписание**: раз в день (86400 секунд), но можно форсировать через `run_now`

## Feed Agent

**feed-agent** — это LLM-агент, который:

1. Анализирует XML-фид перед загрузкой
2. Принимает решения:
   - Какой `MIN_DISCOUNT` установить (адаптируется под качество фида)
   - Сколько товаров на категорию
   - Какие категории включить/исключить
3. Обнаруживает аномалии (исчезновение категорий, падение скидок)
4. Запускает `update_products.py` с оптимальными параметрами
5. Пересобирает фиды (`npm run build`)
6. Уведомляет других агентов (content, seo) о новых товарах

### Почему агент лучше скрипта

| | Скрипт (cron) | Feed Agent |
|---|---|---|
| Анализ фида | Нет | LLM анализирует структуру и качество |
| Адаптация MIN_DISCOUNT | Фиксированный 30% | Адаптируется под фид (20-50%) |
| Обнаружение аномалий | Нет | "Средняя скидка electronics упала с 45% до 28%" |
| Новые категории | Требует ручного добавления | Авто-обнаружение и добавление |
| Уведомления | Нет | Ставит run_now другим агентам |
| Пересборка фидов | Ручная | Автоматическая после обновления |

### Как feed-agent принимает решения

```
Анализ фида:
- Всего офферов: 150,000
- Категорий найдено: 8
- Средняя скидка: 42%
- electronics: 5000 товаров, средняя скидка 35%
- clothing: 3000 товаров, средняя скидка 48%
- shoes: 1500 товаров, средняя скидка 52% ← НОВАЯ!
- beauty: 200 товаров, средняя скидка 15% ← МАЛО!

Решения LLM:
- MIN_DISCOUNT = 30 (хорошее качество фида)
- PRODUCTS_PER_CATEGORY = 200
- Включить: electronics, clothing, shoes, home, auto
- Исключить: beauty (слишком мало товаров, низкие скидки)
- Уведомить: content-agent (новые товары), seo-agent (новая категория)
```

## Категории товаров

Категории определяются **динамически** из фида — нет захардкоженного списка!

### Как это работает

1. Скрипт `update_products.py` парсит фид и собирает **все категории**, которые есть в данных
2. Каждая категория получает имя и иконку из `CATEGORY_META` (если известна) или auto-generated (`capitalize` + `Package`)
3. `categories.json` генерируется автоматически — только из категорий с реальными товарами

### Текущие категории (пример)

| Категория | Название | Иконка |
|-----------|----------|--------|
| electronics | Электроника | Monitor |
| clothing | Одежда | Shirt |
| home | Дом | Home |
| auto | Авто | Car |
| jewelry | Украшения | Gem |

### Если в фиде появятся новые категории

Например, если завтра в фиде появятся товары `shoes` (обувь):
- Скрипт автоматически найдёт их (через `CATEGORY_MAP`)
- Добавит в `products.json`
- Сгенерирует `categories.json` с новой категорией `{id: "shoes", name: "Обувь", icon: "Footprints"}`
- При следующей сборке фидов — категория появится в UI

### Если категория исчезла из фида

Например, если товары `jewelry` закончились:
- Скрипт не найдёт товары этой категории
- Удалит её из `categories.json`
- При следующей сборке — категория исчезнет из UI

### Добавление новых категорий

Чтобы добавить поддержку новой категории из фида:

1. Добавить маппинг `cat_id → category_name` в `CATEGORY_MAP` в `update_products.py`
2. Добавить метаданные в `CATEGORY_META` (название и иконка) — опционально, но красиво
3. Запустить обновление: `python3 scripts/update_products.py`

Пример:
```python
# CATEGORY_MAP — добавить маппинг ID категории фида
"201999001": "toys",  # ID из фида → наша категория

# CATEGORY_META — добавить отображение
"toys": {"name": "Игрушки", "icon": "ToyBrick"},
```

## Как обновлять товары

```bash
# Обновить товары из Admitad feed
cd /opt/smart-skidka-agents
python3 scripts/update_products.py

# Пересобрать фиды и статические страницы
cd /var/www/dealshub-miniapp/v2
npm run build

# Скопировать сборку в корень сайта
cp -r dist/* /var/www/dealshub-miniapp/
```

## Как агенты взаимодействуют с товарами

### Разрешённые поля для изменения (whitelist)

Агенты могут изменять только эти поля в `products.json`:

- `description` — описание товара
- `badge` — бейдж (Тренд, ХИТ, NEW)
- `priority` — приоритет сортировки
- `discount` — размер скидки
- `promo_code` — промокод
- `expires_at` — срок действия акции

### Защищённые поля (нельзя менять)

- `id`, `itemId`, `title`, `category`, `price`, `oldPrice`, `image`, `affiliateLink`, `aliLink`, `rating`, `orders`

### Действия агентов

| Агент | Действия | Приоритет |
|-------|----------|-----------|
| **feed** | Анализ фида, обновление товаров, пересборка фидов, уведомления | -1 (первый) |
| seo | Обновление meta-тегов в index.html | 1 |
| smm | Постинг в Telegram канал | 1 |
| performance | Добавление бейджей, приоритизация товаров | 1 |
| analytics | Анализ трендов, приоритизация топ-товаров | 2 |
| email | Приоритизация товаров из email-рассылок | 3 |
| content | Создание категорийных страниц, обновление описаний, блог-посты | 4 |
| trend | Анализ трендов, рекомендации другим агентам | 0 |

## Квоты

- **Новые страницы категорий**: максимум 10 в сутки
- **Посты в Telegram**: максимум 10 в день, cooldown 30 минут
- **Файловые операции**: максимум 5 за один запуск агента

## Сборка фидов

Скрипт сборки v2 (`v2/package.json`):

```bash
tsc -b && vite build \
  && node scripts/generate-product-pages.js \
  && node scripts/generate-category-pages.js \
  && node scripts/generate-sitemap.js \
  && node scripts/generate-turbo.js \
  && node scripts/generate-yml.js
```

Это генерирует все статические HTML-страницы, sitemap, turbo.xml и yandex-market.yml.

## Синхронизация агентов ↔ фиды

После изменения `products.json` агентами необходимо пересобрать фиды:

1. Агент изменяет `products.json` (через `update_item_description`, `add_badge`, `prioritize_products`)
2. Запускается `cd /var/www/dealshub-miniapp/v2 && npm run build`
3. Результат копируется в корень: `cp -r dist/* /var/www/dealshub-miniapp/`

## Контакты и логи

- Логи оркестратора: `journalctl -u sk-agents-orchestrator.service`
- Логи обновления товаров: `/var/log/smartskidka-products.log`
- Дашборд агентов: `http://localhost:8080` (если запущен)
